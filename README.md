# GPS Rover

GPS Rover是一个基于Qt框架的GNSS/IMU导航监控系统，用于移动站实时导航，特别适用于农业播种等场景。

## 项目结构

- **src/core/**：核心功能模块
  - **common/**：通用组件（日志系统、数据类型等）
  - **sensor/**：传感器接口（IMU/GPS数据采集）
  - **estimator/**：状态估计模块（卡尔曼滤波/低通滤波）
  - **controller/**：控制器模块
  - **fileio/**：文件IO操作
- **src/ui/**：用户界面模块
  - **mainwindow/**：主窗口实现
  - **mapwidget/**：地图显示组件
  - **coordinatewidget/**：坐标系统和路线生成组件
- **src/ThirdParty/**：第三方库

## 核心功能

- **实时导航监控**：显示GNSS和IMU融合后的位置、速度、姿态数据
- **历史轨迹回放**：支持播放之前记录的导航轨迹
- **地图可视化**：基于高德地图API，实时显示设备位置和轨迹
- **数据源切换**：支持定位+惯性融合导航和直接使用粗精度GPS数据两种模式
- **支持路线生成**：在底图上支持路线绘制或者生成，按照参数在多边形区域内生成弓型覆盖路线
- **测量功能**：支持在地图上绘制测量线、测量距离、测量面积等功能
- **传感器数据校准**：支持IMU传感器校准功能
- **实时数据显示**：实时显示IMU和GPS的原始数据和处理后的数据

## 技术架构

### 数据流
1. **数据采集层**：通过SensorInterface从IMU和GPS模块采集原始数据
2. **数据处理层**：StateEstimator对传感器数据进行融合处理（卡尔曼滤波或低通滤波）
3. **UI显示层**：MainWindow接收处理后的数据并更新UI显示
4. **地图层**：MapWidget基于高德地图API显示位置和轨迹

### 关键组件
- **SensorInterface**：负责与传感器硬件通信，采集实时数据
- **StateEstimator**：实现传感器数据融合，提供准确的状态估计
- **MainWindow**：管理整个应用的UI和控制逻辑
- **MapWidget**：实现地图显示和交互功能
- **CoordinateWidget**：实现路线生成和坐标系统管理

## 开发说明

- **开发环境**：Windows + MinGW编译器 + Qt框架
- **地图API**：高德地图JS API（key: 65052422c91868483f74aa887e35527e）
- **底层数据解析**：已实现，上层需要实现正常调用以及按需刷新
- **开发原则**：如无必要，勿增实体

## 快速开始

1. **配置串口**：在UI中选择正确的IMU和GPS串口
2. **连接设备**：点击"连接IMU"和"连接GPS"按钮
3. **启动数据采集**：点击"开始"按钮
4. **查看数据**：在主界面查看实时导航数据和地图显示
5. **生成路线**：在坐标系统面板设置参数并生成路线

# GPS-Rover 系统梳理文档

## 1. 项目概述

GPS-Rover 是一套上位机系统，通过传感器采集定位、加速度等实时高频数据，进行解析、计算、编排等处理，将计算结果展示在地图上，并通过串口上的 Modbus 等协议向下位机发送指令。

主要功能：
- 传感器数据采集（IMU、GPS）
- 数据处理与融合
- 地图显示与导航
- 坐标系统管理
- PLC 通信

## 3. 核心模块分析

### 3.1 主窗口模块 (MainWindow)

**主要功能**：
- 初始化系统组件
- 管理传感器连接
- 处理传感器数据
- 更新UI显示
- 协调各模块工作

**核心方法**：
- `initModules()`: 初始化各模块
- `initSignalsSlots()`: 连接信号槽
- `onImuDataUpdated()`: 处理IMU数据更新
- `onGnssDataUpdated()`: 处理GPS数据更新
- `updateMapFromTimer()`: 定时更新地图

### 3.2 传感器模块 (Sensor)

**主要功能**：
- 管理传感器连接
- 采集传感器数据
- 数据缓存和批量发送
- 时间同步

**核心类**：
- `SensorBase`: 传感器基类
- `ImuSensor`: IMU传感器实现
- `GpsSensor`: GPS传感器实现

**数据流程**：
1. 传感器线程采集数据
2. 数据缓存到缓冲区
3. 定时发送缓存数据
4. 主窗口接收并处理数据

### 3.3 状态估计模块 (StateEstimator)

**主要功能**：
- 融合传感器数据
- 估计位置、速度和航向
- 支持卡尔曼滤波和低通滤波
- 坐标转换

**核心方法**：
- `update()`: 更新状态估计
- `updateLowPass()`: 使用低通滤波更新状态
- `updateKalman()`: 使用卡尔曼滤波更新状态
- `geoToENU()`: 经纬度转ENU坐标系

### 3.4 地图更新模块 (MapUpdateThread)

**主要功能**：
- 在独立线程中更新地图
- 避免阻塞主窗口线程
- 处理地图位置和方向更新

**核心类**：
- `MapUpdateThread`: 地图更新线程
- `MapUpdateWorker`: 地图更新工作类

## 4. 函数调用关系

### 4.1 程序启动流程

1. `main()` → `MainWindow` 构造函数
2. `MainWindow` 构造函数 → `initModules()`
3. `initModules()` → 初始化传感器线程、状态估计器、地图更新线程
4. `initSignalsSlots()` → 连接信号槽
5. `MainWindow::show()` → 显示主窗口

### 4.2 传感器数据流程

1. 传感器线程启动 → 采集数据
2. 数据缓存 → 定时发送
3. 主窗口接收数据 → 处理并更新UI
4. 状态估计器更新状态
5. 地图更新线程更新地图显示

### 4.3 坐标系设置流程

1. 用户设置O点和A点
2. `on_generateConfigButton_clicked()` → 生成配置
3. `TableWidget::setOriginAndDirection()` → 设置原点和方向
4. `TableWidget::generateBlocks()` → 生成小区参数
5. `MapUpdateThread::setMapOriginAndDirection()` → 更新地图

## 5. 多线程调用关系

### 5.1 传感器线程

- **IMU线程**：采集IMU数据，缓存并定时发送
- **GPS线程**：采集GPS数据，缓存并定时发送，处理时间同步

### 5.2 地图更新线程

- **MapUpdateThread**：在独立线程中处理地图更新，避免阻塞主窗口

### 5.3 线程间通信

- 使用Qt信号槽机制进行线程间通信
- 传感器线程 → 主窗口：发送传感器数据
- 主窗口 → 地图更新线程：发送地图更新请求

## 6. 信号传递机制

### 6.1 传感器信号

- `ImuSensor::imuDataReceived`: IMU数据接收信号
- `GpsSensor::gnssDataReceived`: GPS数据接收信号
- `GpsSensor::timeSyncCompleted`: 时间同步完成信号

### 6.2 地图更新信号

- `MapUpdateThread::updateMapPositionSignal`: 更新地图位置信号
- `MapUpdateThread::updateMapOrientationSignal`: 更新地图方向信号
- `MapUpdateThread::setMapOriginAndDirectionSignal`: 设置地图原点和方向信号
- `MapUpdateThread::setCoordinateParamsSignal`: 设置坐标系参数信号

### 6.3 UI信号

- 按钮点击信号
- 下拉框选择信号
- 定时器超时信号

## 7. 系统流程

### 7.1 数据采集流程

1. 用户连接传感器
2. 传感器线程启动
3. 传感器采集数据
4. 数据缓存和批量发送
5. 主窗口接收并处理数据
6. 状态估计器更新状态

1. GPS传感器获取时间信息
2. 计算时间同步偏移量
3. 发送时间同步完成信号
4. IMU传感器更新时间同步状态
5. 系统使用同步后的时间

### 7.3 坐标系设置流程

1. 用户设置O点和A点
2. 生成坐标系配置
3. 更新地图显示
4. 系统使用新的坐标系

## 8. 异常处理

- 传感器连接异常处理
- 数据更新异常处理
- 线程异常处理
- 网络连接异常处理

## 9. 性能优化建议

1. **数据缓存优化**：合理设置缓存大小，避免内存占用过高
2. **线程管理优化**：确保线程正确启动和停止，避免资源泄漏
3. **信号槽连接优化**：使用合适的连接类型，避免线程阻塞
4. **数据处理优化**：减少不必要的数据拷贝和计算
5. **UI更新优化**：限制UI更新频率，避免UI卡顿

## 11. 总结

GPS-Rover 系统是一个复杂的上位机系统，包含多个模块和线程，通过传感器采集数据，进行处理和融合，最终展示在地图上并与下位机通信。系统采用了模块化设计，使用了多线程技术和信号槽机制，确保了数据的实时性和系统的稳定性。

通过本梳理文档，希望能够帮助开发者更好地理解系统架构和工作原理，从而更有效地进行系统维护和扩展。

# GPS-Rover 项目总手册
## 1. 项目概述
GPS-Rover 是一个基于 Qt 框架的导航监控系统，主要用于实时显示 GNSS 和 IMU 融合后的位置、速度、姿态数据，支持历史轨迹回放、地图可视化、数据源切换、路线生成和测量功能。

### 核心功能
- 实时导航监控：显示 GNSS 和 IMU 融合后的位置、速度、姿态数据
- 历史轨迹回放：支持播放之前记录的导航轨迹
- 地图可视化：支持多种底图切换，实时显示设备位置和轨迹
- 数据源切换：支持定位+惯性融合导航和直接使用粗精度 GPS 数据两种模式
- 支持路线生成：在底图上支持路线绘制或者生成等，按照参数在多边形区域内生成弓型覆盖路线
- 测量功能：支持在地图上绘制测量线、测量距离、测量面积等功能
## 2. 系统架构
项目采用分层架构设计，主要包括以下层次：

### 2.1 底层数据解析库
- SerialPort ：串口通信基础类，负责底层串口的打开、关闭、读写操作
- GnssSerialLoader ：GNSS 数据加载器，负责从串口读取 GNSS 数据并解析 NMEA 协议
- ImuSerialLoader ：IMU 数据加载器，负责从串口读取 IMU 数据并解析二进制协议
### 2.2 核心传感器管理
- SensorDataManager ：传感器数据管理中心，采用单例模式，负责统一管理传感器数据，提供简洁的数据订阅机制
- SensorInterface ：传感器接口类，负责与后端传感器通信和数据获取
- DataSourceManager ：数据源管理类，负责管理不同类型的数据源
### 2.3 状态估计和控制
- StateEstimator ：状态估计模块，负责估计设备的位置、速度、姿态等状态
- PredictiveController ：预测触发控制器，负责预测设备的运动轨迹并触发相应的控制动作
### 2.4 地图和路径
- FieldManager ：田块管理模块，负责管理田块信息
- PathGenerator ：路径生成模块，负责生成设备的运动路径
### 2.5 坐标转换
- CoordinateConverter ：坐标转换模块，负责不同坐标系之间的转换
### 2.6 数据库
- DBManager ：数据库管理模块，负责管理设备的历史数据
### 2.7 UI 界面
- MainWindow ：主窗口类，负责整个应用程序的用户界面和控制逻辑
## 3. 核心功能模块
### 3.1 底层串口通信（SerialPort）
- 支持 Windows 和 Linux 平台
- 提供串口的打开、关闭、读写操作
- 支持设置波特率、数据位、停止位、校验位等参数
### 3.2 GNSS 数据解析（GnssSerialLoader）
- 支持解析 NMEA 协议的 GNSS 数据
- 支持解析 GGA、RMC、VTG、GSA、GSV、GST、ZDA 等 NMEA 消息
- 提供位置、速度、航向、DOP 值、可见卫星数等信息
- 支持时间同步和 GPS 周数计算
### 3.3 IMU 数据解析（ImuSerialLoader）
- 支持解析二进制协议的 IMU 数据
- 提供加速度、角速度、角度、磁场、温度等信息
- 支持数据过滤和异常处理
- 支持计算磁航向
### 3.4 传感器数据管理（SensorDataManager）
- 采用单例模式，提供统一的数据管理接口
- 支持数据订阅机制，允许不同模块订阅不同类型的数据
- 支持数据源切换，包括直接 GNSS 数据、KF-GINS 融合数据和模拟数据
- 提供原始数据访问接口
### 3.5 数据融合（FusionManager）
- 预留接口，用于实现 IMU 和 GNSS 数据的融合
- 支持设置融合启用/禁用状态
- 提供融合后数据的访问接口
### 3.6 状态估计（StateEstimator）
- 负责估计设备的位置、速度、姿态等状态
- 支持 IMU 数据校准
### 3.7 路径生成（PathGenerator）
- 支持在多边形区域内生成弓型覆盖路线
- 支持设置路径参数，如行距、重叠率等
### 3.8 地图可视化
- 支持多种底图切换
- 实时显示设备位置和轨迹
- 支持在地图上绘制测量线、测量距离、测量面积等
## 4. 数据流程
### 4.1 数据流程概述
1. 底层串口开启 ：通过 SerialPort 类打开 GNSS 和 IMU 设备的串口
2. 数据读取、解析 ：通过 GnssSerialLoader 和 ImuSerialLoader 从串口读取数据并解析
3. 数据订阅与处理完毕信号 ：通过 SensorDataManager 管理数据并分发给订阅者
4. 应用层 ：通过 MainWindow 接收数据并在 UI 上显示
### 4.2 详细数据流程
1. 串口初始化 ：在 SensorDataManager 初始化时，创建 GnssSerialLoader 和 ImuSerialLoader 实例，打开串口
2. 数据采集 ：在 SensorDataManager::processSensorData() 中，定期从加载器获取数据
3. 数据处理 ：对获取的数据进行处理，如时间同步、数据过滤等
4. 数据分发 ：通过 notifySubscribers() 方法通知订阅者，并通过信号槽机制发送数据更新信号
5. 数据融合 ：在 FusionManager 中对 IMU 和 GNSS 数据进行融合
6. 状态估计 ：在 StateEstimator 中估计设备的状态
7. UI 显示 ：在 MainWindow 中接收数据并在 UI 上显示

## 6. 存在的问题和瓶颈
### 6.1 数据流程问题
- 数据流程冗余 ：SensorDataManager 和 SensorInterface 功能重叠，导致数据流程复杂
- 信号槽连接混乱 ：存在过多的信号槽连接，导致代码难以维护
- 数据融合功能未实现 ：FusionManager 仅为预留接口，未实现具体的融合算法
### 6.2 错误处理问题
- 错误处理不完善 ：部分模块的错误处理不够完善，可能导致系统崩溃
- 异常处理不当 ：部分地方使用 try-catch 捕获异常但未做任何处理
### 6.3 性能问题
- 定时器使用不当 ：多个定时器同时运行，可能导致性能下降
- 数据处理效率低 ：部分数据处理逻辑效率较低，如字符串拼接、数据拷贝等
### 6.4 代码结构问题
- 代码结构混乱 ：部分模块的代码结构不够清晰，职责不够明确
- 命名不规范 ：部分变量和函数的命名不够规范，影响代码可读性
- 注释不完整 ：部分代码缺少注释，影响代码可维护性
### 6.5 依赖问题
- 依赖管理不当 ：第三方库的依赖管理不够完善
- 条件编译处理不当 ：对可选依赖的处理不够完善，可能导致编译错误

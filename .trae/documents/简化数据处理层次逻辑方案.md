# 简化数据处理层次逻辑方案

## 现状分析

当前GPS-Seeder项目的数据处理流程存在以下问题：

1. **层次过多**：数据从串口设备到应用层经过了多个类的处理，包括SerialPort → ImuSerialLoader/GnssSerialLoader → SensorInterface → DataSourceManager → 应用层
2. **调试复杂**：数据流转经过多个环节，难以追踪和调试
3. **模块职责不够清晰**：SensorInterface既要管理串口连接，又要处理数据，还要管理数据源
4. **订阅机制不够灵活**：上层应用需要通过多个信号来获取不同类型的数据

## 简化方案

### 核心设计思想

采用**统一数据管理中心**模式，结合**观察者模式**实现简洁的数据订阅机制，减少中间层次，简化数据流转路径。

### 具体实现方案

#### 1. 创建统一数据管理中心 (SensorDataManager)

- **核心功能**：
  - 管理所有传感器的串口连接
  - 处理数据的读取和解析
  - 维护数据源状态
  - 统一分发数据给订阅者

- **设计特点**：
  - 单例模式，确保全局唯一
  - 内部多线程处理串口数据，避免阻塞主线程
  - 提供简洁的API接口

#### 2. 实现数据订阅机制

- **观察者模式**：
  - 定义数据类型枚举（IMU、GNSS、FUSION等）
  - 提供订阅/取消订阅接口
  - 当数据更新时，自动通知所有订阅者

- **订阅方式**：
  - 基于回调函数的订阅
  - 基于信号槽的订阅（Qt风格）

#### 3. 简化API接口

- **初始化接口**：
  ```cpp
  bool initialize(const QString &imuPort, int imuBaudrate, const QString &gnssPort, int gnssBaudrate);
  ```

- **数据源控制**：
  ```cpp
  bool setDataSource(DataSourceType type);
  DataSourceType getCurrentDataSource() const;
  ```

- **数据订阅**：
  ```cpp
  void subscribeData(DataType type, std::function<void(const void*)> callback);
  void unsubscribeData(DataType type);
  ```

- **状态查询**：
  ```cpp
  bool isConnected() const;
  bool isImuConnected() const;
  bool isGnssConnected() const;
  ```

#### 4. 集成现有功能

- **保留KF-GINS核心功能**：继续使用ImuSerialLoader和GnssSerialLoader进行数据解析
- **保留数据源管理**：支持GNSS_DIRECT、KF_FUSION、SIMULATION三种数据源
- **保留原始数据输出**：用于调试和故障排除

#### 5. 上层应用使用方式

```cpp
// 初始化数据管理器
SensorDataManager::instance()->initialize("COM3", 115200, "COM4", 9600);

// 订阅IMU数据
SensorDataManager::instance()->subscribeData(DataType::IMU, [](const void* data) {
    const IMU* imuData = static_cast<const IMU*>(data);
    // 处理IMU数据
});

// 订阅GNSS数据
SensorDataManager::instance()->subscribeData(DataType::GNSS, [](const void* data) {
    const GNSS* gnssData = static_cast<const GNSS*>(data);
    // 处理GNSS数据
});

// 设置数据源
SensorDataManager::instance()->setDataSource(DataSourceType::KF_FUSION);

// 启动数据采集
SensorDataManager::instance()->start();
```

## 优势

1. **简化层次**：减少了中间处理层次，数据流转更加直接
2. **易于调试**：统一的数据管理中心使得数据追踪和调试更加容易
3. **灵活订阅**：上层应用可以根据需要订阅特定类型的数据
4. **职责清晰**：每个模块只负责自己的核心功能
5. **易于扩展**：新的传感器或数据源可以轻松集成到系统中

## 实现步骤

1. **创建SensorDataManager类**：实现统一数据管理中心
2. **实现观察者模式**：添加数据订阅和分发功能
3. **集成现有功能**：整合KF-GINS的串口数据处理能力
4. **修改上层应用**：更新MainWindow等使用新的API
5. **测试和验证**：确保所有功能正常工作

## 预期效果

通过这个简化方案，上层应用可以通过非常简洁的代码来获取和使用传感器数据，同时保持系统的灵活性和可扩展性。调试过程也会变得更加简单，因为数据流转路径更加清晰。
